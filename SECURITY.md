## 报告漏洞

如果你发现了安全漏洞,请**不要**公开提交 Issue。

### 报告流程

1. **私密报告**: 发送邮件到项目维护者(或使用 GitHub Security Advisory)
2. **提供信息**:
   - 漏洞描述
   - 复现步骤
   - 影响范围
   - 可能的修复方案(如果有)
3. **等待响应**: 我们会在 48 小时内回复

### 我们的承诺

- 及时确认收到报告
- 评估漏洞严重程度
- 开发和测试修复方案
- 发布安全更新
- 在修复后公开致谢(如果你同意)

## 安全最佳实践

### 使用 AutoClaw 时

1. **权限管理**
   - 只授予必要的权限
   - 定期检查应用权限

2. **脚本安全**
   - 不要运行来源不明的脚本
   - 审查市场任务的代码
   - 避免在脚本中硬编码敏感信息

3. **数据保护**
   - 定期备份重要数据
   - 不要在任务中存储密码
   - 使用加密存储敏感配置

4. **网络安全**
   - 使用 HTTPS 连接
   - 验证 API 响应
   - 避免在公共网络运行敏感任务

### 开发者注意事项

1. **代码审查**
   - 所有 PR 需要代码审查
   - 使用静态代码分析工具

2. **依赖管理**
   - 定期更新依赖
   - 检查依赖的安全漏洞

3. **输入验证**
   - 验证所有用户输入
   - 防止代码注入
   - 使用参数化查询

## 已知安全考虑

### eval() 使用
- **位置**: `task_executor.js:62`
- **风险**: 代码注入
- **缓解**: 仅执行用户明确添加的脚本,建议使用沙箱环境
- **计划**: 未来版本将使用更安全的执行方式

### 数据存储
- **方式**: AutoX.js storages (本地存储)
- **加密**: 当前未加密
- **建议**: 不要存储敏感信息
