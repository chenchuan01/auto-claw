# 代码规范

## 符号使用规范

### 禁止使用 Emoji

**严格禁止**在代码、UI文本、注释、文档中使用任何Emoji符号。

### 允许使用的符号

使用以下字体符号替代Emoji:

| 用途 | 允许的符号 |
|------|-----------|
| 设置 | `[S]` 或 `[*]` |
| 数据/存储 | `[D]` |
| 其他/选项 | `[O]` |
| 成功/OK | `[OK]` |
| 错误/失败 | `[X]` 或 `[!!]` |
| 执行/播放 | `>` 或 `[>]` |
| 添加 | `+` |
| 编辑 | `[E]` |
| 删除 | `[-]` |
| 返回/后退 | `<` |
| 刷新 | `R` |
| 帮助 | `?` |
| 警告 | `!` |
| 菜单 | `= ` |
| 列表 | `- ` |
| 空状态 | `o` |
| 云/网络 | `@` |

### 示例

```javascript
// 错误
toast('✅ 操作成功');
text.setText('⚙️ 设置');

// 正确  
toast('[OK] 操作成功');
text.setText('[*] 设置');
```

### 按钮文本规范

```javascript
// 错误
'<button text="📦 保存"/>'

// 正确
'<button text="[S] 保存"/>'
```

### 状态提示规范

```javascript
// 错误
statusText.setText('⚠️ 请输入API Key');

// 正确
statusText.setText('[!] 请输入API Key');
```

## 代码风格

- 使用ES5语法 (AutoX.js兼容)
- 模块化开发，避免单个文件超过300行
- 使用四个空格缩进
- 变量和函数使用驼峰命名
- 常量使用大写下划线命名

## 模块完整性与引用检查

### 修改原则

修改代码时，必须检查：

1. **模块依赖**：确认所有 `require` 导入的模块都是存在且正确的
2. **方法调用**：确认调用的方法在目标模块中确实存在
3. **事件绑定**：如果从布局中移除了UI组件，必须同步删除对应的事件绑定代码
4. **ID一致性**：布局中定义的 `id` 必须与代码中 `ui.id` 引用完全一致
5. **文件完整性**：写入文件时必须保证完整写入，不能出现文件内容为空或不完整的情况

### 常见错误预防

- 删除UI元素后，必须同步删除对应的事件绑定代码（如 `.on('click', ...)`）
- 修改模板变量时，遵循 AutoX.js 模板语法：直接使用 `{{variable}}`，不要加 `this.` 前缀
- 使用 AutoX.js API 时，注意方法名正确性：如 `getParent()` 而非 `parent()`
- 新增模块后，必须在 `ui_manager.js` 中正确导入和实例化

### 验证步骤

每次修改后，检查以下项目：
- [ ] 所有导入的模块文件都存在
- [ ] 所有调用的方法都存在
- [ ] 所有引用的 UI ID 都在布局中定义
- [ ] 删除元素后没有残留的事件绑定
- [ ] 新建文件内容完整，不为空
